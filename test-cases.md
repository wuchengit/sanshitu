## 代码拆分建议

`sanshitu.html` 当前 1085 行，建议拆分为三文件：

### 方案 A（推荐）：三文件分离

| 文件 | 内容 | 估计行数 | 说明 |
|------|------|----------|------|
| `sanshitu.html` | HTML 骨架 + CSS | ~500 行 | 内联 CSS 保留（SPA 场景避免 FOUC）|
| `sanshitu.js` | 所有 JS 逻辑 | ~600 行 | 拆模块：初始化/渲染/生成/翻译/历史/预设 |
| `sanshitu.css` | 样式 | ~400 行 | 按模块分组：布局/卡片/网格/模态框/工具条 |

### 方案 B（轻量）：JS 外置

| 文件 | 内容 | 说明 |
|------|------|------|
| `sanshitu.html` | HTML + CSS + 内联初始化 | CSS 保留，只把 JS 搬出去 |
| `sanshitu.js` | 所有 JS | 最省事的拆分 |

### 拆分注意事项
1. 确保 `defer` 加载 JS，避免 DOM 未准备好
2. CSS 变量和 暗色模式变量保持集中管理
3. API 地址等常量提为 `const` 放在 JS 头部
4. 版本更新检测不要漏掉新文件
5. 拆分后需要一个 HTML + 两个（或一个）额外 HTTP 请求，Nginx 无缓存问题

## 测试页面设计

建议创建一个独立的测试运行页面 `/tests.html`：

### 页面功能
- [ ] 测试用例树状列表显示（按模块折叠）
- [ ] 每条用例有：ID / 名称/ 步骤 / 预期 / 实际结果 / 状态（通过/失败/跳过）
- [ ] 支持逐条手动验证打勾
- [ ] 支持一键全部重置
- [ ] 导出测试报告（JSON/Markdown）
- [ ] 测试进度条（X/39 通过）

### 页面位置
`/usr/local/lighthouse/softwares/wordpress/tests.html`

### Nginx 路由
```
location = /tests { try_files /tests.html =404; }
```
